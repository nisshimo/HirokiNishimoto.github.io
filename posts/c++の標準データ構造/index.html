C++の標準データ構造

2019.10.14
アルゴリズム c++

stack

LIFOのデータ構造

メンバ関数

  • push() : 要素の挿入
  • top() : 先頭の要素の参照
  • pop() : 先頭の要素の破棄
  • size() : 現在の要素数

計算コストは全てO(1)

練習問題

queue

FIFOのデータ構造

メンバ関数

  • push() : 要素の挿入
  • front() : 先頭の要素の参照
  • pop() : 先頭の要素の破棄
  • size() : 現在の要素数

計算コストは全てO(1)

練習問題

priority_queue

挿入されたデータの中で最大の要素の参照が低数時間でできる。 実装には2分ヒープなど

メンバ関数

  • push() : 要素の挿入
  • top() : 先頭の要素の参照
  • pop() : 先頭の要素の破棄
  • size() : 現在の要素数

計算コストはtop() : O(1), push(), pop() : O(logN)

練習問題

set

集合を表現するライブラリ

種類

要素の重複を許さないもの

  • set : データに順序をつけて管理

    • 通常２分探索木で実装される
    • 挿入や探索0(ligN)

  • unorderd_set : データの順序を無視して管理

    • 通常ハッシュテーブルで実装される
    • 各操作は平均的には低数時間で収まると期待される

要素の重複を許すもの

  • multiset
  • unorderd_multiset

メンバ関数

  • insert(element) : 要素の挿入
  • count(element) : 要素の数を数える（要素の重複を許さないsetなら0 or 1)
  • size() : 現在の要素数
